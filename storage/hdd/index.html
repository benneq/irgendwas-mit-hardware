LMR (Longitudinal Magnetic Recording)

Ursprüngliche Technologie, bei der die Bits parallel zur Rotationsrichtung des Datenträgers liegen.

PMR (Perpendicular Magnetic Recording)

Um die Speicherdichte zu erhöhen, werden die Bits senkrecht zur Oberfläche des Datenträgers angeordnet.

Durch die Einfühung von SMR Festplatten, die ebenfalls auf PMR setzen, wurden die nicht-SMR Festplatten zu CMR umbenannt.

CMR (Conventional Magnetic Recording)

Mit der Einführung von SMR Festplatten, gab es zwei unterschiedliche Technolgien, die auf senkrechte Bits setzten. Um deren Unterscheidung zu ermögilchen wurden die ursprünglichen PMR Festplatten in CMR umbenannt.

SMR (Shingled Magnetic Recording)

Um die Speicherdichte von PMR Festplatten noch weiter zu erhöhen, werden überlappende Spuren verwendet. Dadurch leidet die Schreibperformance.

Es gibt 3 Varianten, die unterschiedliche Anforderungen an das Host System stellen und unterschiedliche Performance Charakteristika aufweisen.

DM-SMR (Device Managed Shingled Magnetic Recording)

Bei DM-SMR kümmert sich die Festplatte selbst um die Organisation der Daten. Das Host System weiß in diesem Fall nichts von der internen Arbeitsweise der Festplatte.

Vorteile: Plug & Play, weil das Host System nicht wissen muss wie die Festplatte intern arbeitet.

Nachteile: Keine Kontrolle durch das Host System möglich, dadurch unvorhersehbare Performance, und somit ungeeignet für Enterprise Anwendungen.

HM-SMR (Host Managed Shingled Magnetic Recording)

Das Host System übernimmt das Management der Datenströme, Lese- und Schreiboperationen und der Zonen.

Vorteile: Kontrolle über die Performance.

Nachteile: Der Host muss SMR unterstüzten.
